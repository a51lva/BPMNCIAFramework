Change Patterns Catalog

Activity Differences

An activity exists in one process, but no activity representing an equivalent exists in the other process.

An activity exists in one process and an activity that has the same effect exists in the other process.

An activity exists in one process, but represented by a collection of activities in the other process.

Two processes have corresponding collections of activities, if a collection of activities in one process is equivalent to a collection of activities in the other process.

An activity doesn't exist in the first process, but exists in the updated process.

Artifacts Differences

An activity has an artifact reference in one process, but no artifacts reference representing an equivalent exists in the other process.

An activity does not have an artifact reference in the first process, but exists an artifact reference in the updated process.

Authorization Differences

An activity is assigned to one role in one process and to another in the other.

An activity is assigned to a single role in one process and assigned to multiple roles in the other.

An activity is performed by one collection of roles in one process and to another collection of roles in the other process.

Control-flow Differences

Equivalent activities from two processes have different dependencies on which they depend.

A special case of having different dependencies is the case in which one set of activities includes the other.

Another special case of having different dependencies is the case in which the sets of activities are disjoint.

An activity is part of a loop in one process while it is not in the other process.

Gateway Differences

A gateway exists in one process, but no gateway representing an equivalent exists in the other process.

A gateway does not exist in the first process, but exists in the updated process.
